## 1. Executive Summary

### 1.1 Product Overview

TekBreed is a comprehensive AI-powered software engineering learning platform that combines expert-curated content, live instruction, and community collaboration to transform aspiring developers into job-ready engineers. Unlike traditional Massive Open Online Courses (MOOCs) or coding bootcamps, TekBreed offers a hybrid model providing:

- **Structured Learning Paths**: Clear progression from beginner to professional
- **24/7 AI Tutoring**: Personalized assistance adapting to individual learning styles
- **Live Interactive Classrooms**: Real-time instruction with collaborative tools
- **Community-Driven Learning**: Team projects, automated code reviews, and mentorship
- **Career Integration**: Direct pathway from learning to employment

### 1.2 Problem Statement

The global software engineering education market faces four critical challenges:

**1. Tutorial Hell ($15B problem)**  
67% of self-taught learners never complete their learning journey. They consume dozens of disconnected tutorials without a structured path, leading to frustration and abandonment. The average self-learner watches 40+ tutorials before giving up, wasting both time and motivation.

**2. Isolation & Support Gap ($8B problem)**  
73% of online learners cite "lack of support" as their primary reason for failure. Traditional online courses offer no real-time assistance, leaving learners stuck on simple problems for hours or days. Bootcamps provide support but at $15,000+ price points that exclude most potential learners.

**3. Outdated Content ($12B problem)**  
Software engineering evolves rapidly, with major framework updates every 6-12 months. Most online content is 2+ years old, teaching deprecated patterns and obsolete libraries. Learners unknowingly invest time in outdated skills, reducing employability.

**4. Broken Career Pipeline ($20B problem)**  
Despite 3.5 million unfilled US software jobs and $120,000 median salaries, 91% of self-taught learners never reach employability. Traditional CS degrees require 4 years and $100,000+. Bootcamps compress learning into 3-6 months at $15,000+ but maintain 40-60% completion rates and limited post-graduation support.

### 1.3 Solution Overview

TekBreed addresses these challenges through:

1. **Structured Curriculum**: Article â†’ Tutorial â†’ Course â†’ Program progression eliminates tutorial hell
2. **AI Learning Companion**: 24/7 personalized tutoring at scale for <$30/month
3. **Live Hybrid Model**: Combines self-paced learning with live instruction and community support
4. **Always-Current Content**: Quarterly updates, expert-curated, modern best practices
5. **Career Integration**: Job board, portfolio showcasing, and employer connections built-in

## 5. Product Architecture

### 5.1 High-Level System Architecture

TekBreed is built as a modern, cloud-native platform with the following layers:

**Presentation Layer**:
- Web Application (React 19, React Router v7, TypeScript)
- Mobile Application (React Native, planned Q3 2026)
- Progressive Web App capabilities (offline reading, push notifications)

**Application Layer**:
- REST API (Node.js, Express)
- WebSocket Server (Socket.io for real-time features)
- Background Job Processors (Upstash Redis)

**AI/ML Layer**:
- AI Companion Service [Anthropic (Claude Haiku/Sonnet) integration]
- Embedding Service (Upstash Vector/Voyage AI for vector generation and semantic search)
- RAG (Retrieval-Augmented Generation) Pipeline
- Recommendation Engine (Anthropic)

**Data Layer**:
- SQLite (primary database for transactional data, hosted on Turso)
- Redis (caching, sessions, rate limiting, real-time data)
- Upstash Vector (vector database for AI features)
- Bunny CDN

**Infrastructure Layer**:
- Railway (hosting)
- Turso (managed SQLite)
- Bunny (CDN)
- Cloudflare (DNS, security)

**Content Types Hierarchy**:

### 5.3 User Progress Tracking

**Progress Granularity**:
- **Content-level**: Overall progress through article, tutorial, or lesson (0-100%)
- **Section-level**: Progress through major sections within content
- **Checkpoint-level**: Specific interactions (code exercises completed, lessons/sub-modules/modules completed)

**Progress Persistence**:
- Auto-save every 30 seconds during active session
- Save on major events (section completion, exercise submission)
- Resume from last position (scroll position preserved)

### 5.4 AI Companion Architecture

**Components**:

1. **Input Validator**: Sanitizes user input, detects spam/abuse
2. **Context Builder**: Gathers relevant context (user progress, lesson content, conversation history)
3. **Cache Checker**: Semantic similarity search for cached responses
4. **Prompt Engineering**: Constructs optimized prompts for LLM
5. **LLM Integration**: Calls Anthropic (Claude Haiku/Sonnet) API
6. **Response Parser**: Validates and formats LLM output
7. **Storage**: Logs interaction for improvement
8. **Feedback Loop**: Collects user ratings, feeds into fine-tuning

**Context Window Management**:
- Max context: 200,000 tokens (Claude Sonnet 3.5)
- Efficient context pruning for long conversations

### 6.3 AI Learning Companion

**1. Debugging Assistance**
- User pastes broken code + error message
- AI analyzes code, identifies issue
- Explains root cause in plain English
- Suggests fix with explanation of why it works
- Offers preventive advice for future

**Example Interaction**:
```
User: "My React component isn't updating when I change state"
[User pastes code]

AI: "I see the issue! You're mutating state directly on line 12:

this.state.count++

In React, state should never be mutated directly. Instead, use setState:

this.setState({ count: this.state.count + 1 })

Why? React compares state by reference, not value. When you mutate directly,
React doesn't detect the change, so it doesn't re-render.

Try this corrected version:
[Provides corrected code]

Want to learn more about React's state management? Check out the 
'State and Lifecycle' lesson."
```

**2. Concept Explanations**
- User asks "What is X?" or "How does Y work?"
- AI explains concept at user's skill level
- Provides analogies, examples, diagrams
- Links to relevant TekBreed content

**5. Interview Preparation**

### 15.1 Glossary

**MAL**: Monthly Active Learners - users who complete at least one learning unit per month

**LTV**: Lifetime Value - total revenue expected from a customer over their relationship

**CAC**: Customer Acquisition Cost - marketing spend divided by new customers acquired

**Churn**: The rate at which customers cancel their subscriptions

**ARPU**: Average Revenue Per User - total MRR divided by number of paid users

**NPS**: Net Promoter Score - measure of customer satisfaction and loyalty

**DAU/MAU**: Ratio of daily active users to monthly active users (stickiness metric)

**Cohort**: Group of users who signed up in the same time period

**MRR**: Monthly Recurring Revenue

**ARR**: Annual Recurring Revenue

**PPP**: Purchasing Power Parity

**SLA**: Service Level Agreement

**WCAG**: Web Content Accessibility Guidelines

**MDX**: Markdown extended with JSX components

**RAG**: Retrieval-Augmented Generation

**CRDT**: Conflict-free Replicated Data Type
